# UI/UX Design Principles

## Goal

Improve the existing graphical interface without changing product functionality unless explicitly requested.

## Principles

- Keep the existing visual identity.
- Improve clarity, spacing, hierarchy, and accessibility.
- Preserve existing user flows.
- Preserve existing content and controls unless explicitly requested.
- Prefer consistency over novelty.

## UI Improvement Boundaries

Allowed:

- spacing alignment
- clearer hierarchy
- consistent component usage
- responsive fixes
- focus state improvements
- accessibility improvements
- visual polish within existing design language

Not allowed without request:

- full redesign
- new screens
- removed controls
- new flows
- new marketing sections
- changed conversion logic
